### Fetching Projects (Public)
```
User navigates to /projects
    в”Ӯ
    в–ј
ProjectsGrid Component mounts
    в”Ӯ
    в”ңв”Җ Calls useProjects() hook
    в”Ӯ
    в–ј
TanStack Query checks cache
    в”Ӯ
    в”ңв”Җ Is data cached? (< 5 mins old)
    в”Ӯ в”ңв”Җ YES вҶ’ Return cached data
    в”Ӯ в”Ӯ      (instant, no API call)
    в”Ӯ в”Ӯ
    в”Ӯ в””в”Җ NO вҶ’ Fetch from Supabase
    в”Ӯ
    в–ј
Supabase SELECT query
    в”Ӯ
    в”ңв”Җ SELECT * FROM projects
    в”ңв”Җ WHERE (RLS allows public read)
    в”ңв”Җ ORDER BY created_at DESC
    в”Ӯ
    в–ј
PostgreSQL executes
    в”Ӯ
    в”ңв”Җ Applies RLS policy
    в”Ӯ (public can read all)
    в”Ӯ
    в”ңв”Җ Returns results
    в”Ӯ
    в–ј
TanStack Query caches result
    в”Ӯ
    в”ңв”Җ Stores in memory
    в”ңв”Җ Sets 5 minute expiry
    в”ңв”Җ Enables background refetch
    в”Ӯ
    в–ј
Component receives data
    в”Ӯ
    в”ңв”Җ Re-renders with projects
    в”ңв”Җ Maps to ProjectCard components
    в”ңв”Җ Applies CSS styles
    в”Ӯ
    в–ј
Framer Motion animates entry
    в”Ӯ
    в””в”Җ Staggered fade-in effect
```

## рҹ”җ Authentication & Authorization Flow

```
User visits /admin/login
    в”Ӯ
    в–ј
AdminLogin page renders
    в”Ӯ
    в”ңв”Җ Form with email + password
    в”Ӯ
    в–ј
User submits credentials
    в”Ӯ
    в–ј
supabase.auth.signInWithPassword()
    в”Ӯ
    в”ңв”Җ Sends to Supabase Auth API
    в”Ӯ
    в–ј
Supabase Auth verifies
    в”Ӯ
    в”ңв”Җ Checks auth.users table
    в”ңв”Җ Bcrypt password verification
    в”ңв”Җ Creates session token (JWT)
    в”Ӯ
    в–ј
Session stored in browser
    в”Ӯ
    в”ңв”Җ localStorage (Supabase default)
    в”ңв”Җ Sent in API headers
    в”Ӯ
    в–ј
Navigation to /admin
    в”Ӯ
    в–ј
AdminLayout checks session
    в”Ӯ
    в”ңв”Җ useAuth() hook
    в”ңв”Җ supabase.auth.getSession()
    в”Ӯ
    в”ңв”Җ Session exists?
    в”Ӯ в”ңв”Җ YES вҶ’ Render admin content
    в”Ӯ в””в”Җ NO вҶ’ Redirect to /admin/login
    в”Ӯ
    в–ј
All API calls include session
    в”Ӯ
    в”ңв”Җ Supabase RLS policies check
    в”ңв”Җ "auth.role() = 'authenticated'"
    в”Ӯ
    в”ңв”Җ Can INSERT/UPDATE/DELETE?
    в”Ӯ в”ңв”Җ YES вҶ’ Allow operation
    в”Ӯ в””в”Җ NO вҶ’ Deny access
    в”Ӯ
    в–ј
Logout
    в”Ӯ
    в”ңв”Җ supabase.auth.signOut()
    в”ңв”Җ Session cleared
    в”ңв”Җ Redirect to /admin/login
```

## рҹ”’ Row Level Security (RLS) Policies

```
Projects Table:
в”ңв”Җ SELECT (Public Read)
в”Ӯ  в””в”Җ true (everyone can read)
в”Ӯ
в””в”Җ INSERT/UPDATE/DELETE (Authenticated Write)
   в””в”Җ auth.role() = 'authenticated'
      (only logged-in admins)

Certificates Table:
в”ңв”Җ SELECT (Public Read)
в”Ӯ  в””в”Җ true (everyone can read)
в”Ӯ
в””в”Җ INSERT/UPDATE/DELETE (Authenticated Write)
   в””в”Җ auth.role() = 'authenticated'

Status Table:
в”ңв”Җ SELECT (Public Read)
в”Ӯ  в””в”Җ true (everyone can read)
в”Ӯ     (status widget on public site)
в”Ӯ
в””в”Җ INSERT/UPDATE/DELETE (Authenticated Write)
   в””в”Җ auth.role() = 'authenticated'
      (only admin can update)

Contacts Table:
в”ңв”Җ INSERT (Public Submit)
в”Ӯ  в””в”Җ true (visitors can submit forms)
в”Ӯ
в”ңв”Җ SELECT (Authenticated Read)
в”Ӯ  в””в”Җ auth.role() = 'authenticated'
в”Ӯ     (only admin sees messages)
в”Ӯ
в””в”Җ UPDATE (Authenticated Update)
   в””в”Җ auth.role() = 'authenticated'
      (admin marks as read)
```

## рҹҺҜ State Management Strategy

```
Global State (Minimal)
в”ңв”Җ TanStack Query cache
в”Ӯ  в”ңв”Җ Projects list
в”Ӯ  в”ңв”Җ Certificates list
в”Ӯ  в”ңв”Җ Status
в”Ӯ  в”ңв”Җ Messages
в”Ӯ  в””в”Җ Auto-manages fetching
в”Ӯ
в””в”Җ Supabase Auth session
   в”ңв”Җ User object
   в”ңв”Җ Session token
   в””в”Җ Auto-managed

Component State (Local)
в”ңв”Җ Form inputs (email, message, etc)
в”ңв”Җ UI state (modal open/close)
в”ңв”Җ Loading states
в””в”Җ Error messages

Benefits:
вң… Minimal prop drilling
вң… Automatic cache invalidation
вң… Background refetch
вң… Optimistic updates
вң… Real-time subscriptions
вң… No Redux/Context boilerplate
```

This architecture ensures:
- **Type Safety**: TypeScript everywhere
- **Performance**: Caching, lazy loading, code splitting
- **Real-time**: Supabase subscriptions
- **Security**: RLS policies, no exposing secrets
- **Scalability**: Separation of concerns, modular design
- **Developer Experience**: TanStack Query, simple hooks API
